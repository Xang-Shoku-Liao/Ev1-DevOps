# Guia de Contribucion

## Proceso de Desarrollo

Para contribuir a este proyecto, por favor sigue estos pasos:

1. Crea una rama nueva desde `develop` siguiendo el formato `feature/nombre` o `hotfix/nombre`.
2. Realiza tus cambios asegurando la calidad del codigo.
3. Documenta cualquier cambio nuevo en el README si es necesario.

## Reglas de Commits

Utilizamos convenciones descriptivas para los mensajes de commit:

- `feat:` para nuevas funcionalidades.
- `fix:` para correccion de errores.
- `docs:` para cambios en la documentacion.

## Pull Requests

Toda mejora debe integrarse mediante un Pull Request hacia la rama `develop`. El equipo revisara el codigo y, una vez aprobado, se realizara el merge.
